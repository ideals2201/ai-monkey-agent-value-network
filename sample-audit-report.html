Sample deliverable

Public-source route audit report format

This sample shows the report structure a buyer receives for one public earning or value route. It uses public-safe placeholder data and does not claim income.

Decision

Route
Example public bounty or listing route
Verdict
HOLD
Reason
Official source exists, but payment/listing acceptance is not yet verified and one eligibility gate remains unresolved.

Evidence gates

  • Official/current source checked.
  • Value signal identified but not yet received.
  • Eligibility and region rules documented.
  • Automation boundary marked.
  • Income counted only after paid_or_usable_value is received and verified.

Safe next action

Proceed only if the buyer can satisfy the public eligibility gate without sending private credentials, identity documents, payment details, or account screenshots. Otherwise, drop or keep on watch.

Output fields

{
  "route": "example",
  "verdict": "HOLD",
  "official_source": "checked",
  "value_signal": "present_not_received",
  "automation_boundary": "human_or_authorized_session_only",
  "paid_or_usable_value": "No",
  "next_action": "resolve eligibility gate"
}
